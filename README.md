# Google Sheets & GAS 통합 제어 프레임워크 (Master-Table System)

본 프로젝트는 구글 스프레드시트를 단순한 데이터 저장소가 아닌, '관리 테이블(Master Table)'을 기반으로 작동하는 중앙 제어형 자동화 프레임워크로 활용합니다. 데이터 무결성 검증, 인메모리(In-memory) 기반 SQL 처리, 그리고 검색 기반 액션 에이전트를 결합하여 확장성 높은 업무 환경을 제공합니다.

---

## 🚀 주요 기술 특징 (Technical Highlights)

본 시스템은 Google Apps Script(GAS) 환경의 구조적 한계를 우회하고, 스프레드시트의 기능을 확장하는 독자적인 최적화 로직을 포함하고 있습니다.

### 1. AlaSQL 기반 인메모리 SQL 엔진
* **관계형 데이터 조작:** HTML UI를 통해 오픈소스 SQL 라이브러리인 **AlaSQL**을 구동합니다. 시트 데이터를 메모리에 로드한 뒤, 표준 SQL 문법(`SELECT`, `INSERT`, `UPDATE`, `DELETE`, `JOIN` 등)으로 시트 데이터를 실시간 조작합니다.
* **고속 처리:** 시트의 개별 셀에 반복적으로 접근하는 대신, 메모리상에서 SQL 연산을 일괄 처리한 뒤 시트에 반영하여 대규모 데이터 처리 속도를 극대화했습니다.

### 2. Zero-Cost 라이프사이클 관리 (Zombie Process Kill)
* **렌더링 기반 감지:** 구글 시트의 `showModelessDialog` 창이 닫힐 때 백그라운드 프로세스가 남는 '좀비 현상'을 해결했습니다. 서버(GAS) 호출 없이 클라이언트 브라우저의 `requestAnimationFrame(rAF)`을 활용한 렌더링 감지 기법을 적용하여, 할당량(Quota) 소모 없이 창이 닫히는 즉시 오디오 및 세션을 물리적으로 종료합니다.

### 3. 병렬 청크(Chunk) 미디어 스트리밍
* **데이터 분할 병렬 로딩:** 대용량 오디오 파일을 Base64로 인코딩한 뒤 청크 단위로 분할 관리합니다. 재생 시 이를 병렬로 호출하여 GAS 환경 특유의 초기 응답 지연(Latency)을 최소화했습니다.

### 4. 커스텀 Opcode 인터프리터
* **추상화된 명령 체계:** 시스템 제어를 단순 함수 호출이 아닌 추상화된 Opcode 단위로 설계하여, 백엔드 로직의 결합도를 낮추고 폭포수 명령체계를 구축하여 토큰 소비를 줄인 뒤 AI 에이전트의 명령 처리 정확도를 높였습니다.

### 5. Webhook 기반 슬랙(Slack) 원격 제어망
* **외부 터미널 연동:** GAS의 Web App 기능을 활용하여 슬랙을 원격 명령 터미널로 구축했습니다. 슬랙에서 발송된 메시지를 Webhook(POST)으로 수신하고 파싱하여 내부 시스템의 트리거로 동작시킵니다.
    (연결을 하실 SLACK 커스텀 명령 URL을 환경설정 탭에 집어넣으시고 GAS 자체를 웹 앱 배포를 하셔야 합니다. AI에게 명령을 보낼때 /명령어 [시트명] 작업지시 평태를 꼭 지켜주십시오)
* **컨텍스트 라우팅 (Context Routing):** 전달된 메시지 내의 '대상 시트명'을 식별자로 추출하여, AI 에이전트가 다수의 시트(분산된 DB) 중 정확한 타겟을 찾아 독립적인 작업을 수행할 수 있도록 라우팅합니다.

### 6. 하이브리드 RAG 및 비용 최적화 파이프라인 (Cross-Platform Indexing)
* **통합 인덱싱 아키텍처:** 구글 드라이브(파일)와 구글 시트(셀 데이터)를 막론하고, 모든 데이터 소스를 단일 텍스트 기반 인덱스로 추상화하여 관리합니다. LLM이 방대한 로우 데이터를 직접 전수 조사할 때 발생하는 과도한 API 과금 및 컨텍스트 제한 문제를 근본적으로 해결했습니다.
* **기계식 프리-필터링 (Keyword-Based Pre-filtering):** 사용자의 명령에서 AI가 도출한 핵심 키워드를 바탕으로, 시스템 스크립트가 인덱스 파일을 먼저 스캔합니다. 시트의 경우에도 전체 범위를 읽지 않고 필터를 거친 유효 데이터만 선별합니다.
* **토큰 효율 극대화:** 사전 필터링을 통해 정제된 최소한의 데이터 컨텍스트만 최종 '작업자 AI(Worker)'에게 전달됩니다. 이를 통해 AI는 불필요한 노이즈 없이 필요한 정보에만 집중할 수 있으며, 전체 시스템의 토큰 소비량을 획기적으로 절감합니다.

## 🤖 액션형 AI 에이전트 (Action-Oriented Agent)

대화가 아닌 **'실제 작업 수행(Execution)'**에 초점을 맞춘 AI 어시스턴트입니다. 

* **기본 원칙 (No Response):** 질문에 대답하는 챗봇이 아닙니다. 명령을 받으면 즉시 대상 시스템(시트, 캘린더, 드라이브)을 제어하며, 작업의 결과물로 응답을 대신합니다.
* **검색 기반 데이터 기입:** 구글 검색기를 가동하여 필요한 외부 최신 데이터를 스스로 수집하고, 지정된 시트의 위치에 자동으로 입력합니다.
* **작업 범위:** 일정 등록, 구글 드라이브 파일 관리, 시트 내 데이터 CRUD 로직 수행.
* **슬랙(Slack) 원격 제어:** 슬랙 메시지를 통해 외부에서 시스템을 제어할 수 있습니다. 
  * ⚠️ **명령 규칙:** 에이전트가 작업할 기준점을 파악할 수 있도록, 명령 메시지에 반드시 **[대상 시트명]**을 명시해야 합니다. (예: "/명령어 [시트명] 에 있는 ~데이터 추가해줘")

## 🏗 시스템 아키텍처 (Architecture)

### 1. 탑다운 관리 테이블 (Top-Down Control)
* 모든 시트의 생성/삭제, 컬럼 명세, 고유 ID 자동 주입 로직은 '관리 테이블'의 설정을 따릅니다.
* **정규식(Regex) 검증:** 관리 테이블에 정의된 정규식 패턴을 통해 데이터 입력 시 무결성을 실시간으로 검증합니다. (예: B3 셀부터 시작되는 데이터 입력 구조 보호)
* **수식 자동 주입:** 신규 행 추가 시, 사전에 정의된 복잡한 함수 수식이 각 셀에 자동으로 주입됩니다.
(예 :=SUM(INDIRECT("Inventory!D"&ROW()),INDIRECT("C"&ROW())))
### 2. 하이브리드 조회 성능 최적화
* GAS 스크립트 실행 속도의 한계를 보완하기 위해, 복잡한 조회 및 필터링 기능은 구글 시트의 순수 함수만으로 동작하도록 설계되어 대규모 데이터 환경에서도 빠른 응답성을 보장합니다.

## 📦 설치 및 설정 (Setup)

1. **업로드:** 제공된 폴더 전체를 본인의 구글 드라이브에 업로드합니다.
2. **시트 복사:** 동봉된 링크로 시트 사본을 생성한 후, 업로드한 구글 드라이브 폴더 내부에 배치합니다.
3. **환경 설정:** 시트를 열고 상단 메뉴의 **[Tools] → [환경설정]**을 클릭하여 필수 키를 입력합니다.
   * `GEMINI_API_KEY`: AI 에이전트 구동용
   * `SLACK_WEBHOOK_URL`: 슬랙 원격 명령 수신용
   * `FOLDER_ID`: 대량 텍스트/이미지 처리 대상 폴더
4. **권한 승인:** 구글 보안 경고 팝업 발생 시 **[고급] → [프로젝트로 이동] → [허용]**을 클릭합니다.
5. **데이터 동기화:** **[관리테이블 저장]** 버튼을 클릭하여 시스템 구조 및 설정값을 캐싱합니다.

(번외) 뮤직 플레이어를 사용 하실려면 우선 MUSIC 폴더에 MP3파일을 집어넣으신뒤 노래준비 버튼을 눌러주세요 해당 작업으로 MUSICLIST 폴더에 분리된 문자열 파일과 해당 목록이 생성됩니다.

## ⚠️ 주의사항
본 시스템은 관리 테이블 구조를 기반으로 작동하므로, 임의의 열 추가나 시트명 변경은 시스템 오류를 유발할 수 있습니다.

모든 구조 변경은 반드시 '관리 테이블'을 통해 진행해 주십시오.

* **베타 버전 안내:** 본 시스템은 지속적으로 고도화 중인 베타 버전입니다. 검증이 완료되지 않은 일부 외부 연동 로직은 공식 지원 범위에서 제외됩니다.
* 모든 구조 변경은 반드시 '관리 테이블'을 통해 진행해 주십시오. 임의의 열 추가나 시트명 변경은 시스템 동기화 오류를 유발할 수 있습니다.
* AI 에이전트를 이용한 대량 작업 전에는 반드시 시트 백업을 권장합니다.
